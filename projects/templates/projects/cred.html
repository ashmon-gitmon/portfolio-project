Credit Card customer usage data¶

Data Source: https://www.kaggle.com/arjunbhasin2013/ccdata

This dataset contains the usage data of few customers of a credit card. My aim is to cluster the users based on their usage behaviour and glean insights from it.

Following is the Data Dictionary for Credit Card dataset :-

CUSTID : Identification of Credit Card holder (Categorical)
BALANCE : Balance amount left in their account to make purchases
BALANCEFREQUENCY : How frequently the Balance is updated, score between 0 and 1 (1 = frequently updated, 0 = not frequently updated)
PURCHASES : Amount of purchases made from account
ONEOFFPURCHASES : Maximum purchase amount done in one-go
INSTALLMENTSPURCHASES : Amount of purchase done in installment
CASHADVANCE : Cash in advance given by the user
PURCHASESFREQUENCY : How frequently the Purchases are being made, score between 0 and 1 (1 = frequently purchased, 0 = not frequently purchased)
ONEOFFPURCHASESFREQUENCY : How frequently Purchases are happening in one-go (1 = frequently purchased, 0 = not frequently purchased)
PURCHASESINSTALLMENTSFREQUENCY : How frequently purchases in installments are being done (1 = frequently done, 0 = not frequently done)
CASHADVANCEFREQUENCY : How frequently the cash in advance being paid
CASHADVANCETRX : Number of Transactions made with "Cash in Advanced"
PURCHASESTRX : Numbe of purchase transactions made
CREDITLIMIT : Limit of Credit Card for user
PAYMENTS : Amount of Payment done by user
MINIMUM_PAYMENTS : Minimum amount of payments made by user
PRCFULLPAYMENT : Percent of full payment paid by user
TENURE : Tenure of credit card service for user

Dealing With outliers¶

Since there are a large number of rows with outliers, instead of removing the rows i replaced the top 10 percentile of values with some value between the 90th and 91st percentile. After doing so, lets make the boxplots again.

In [506]:
#this function kills outliers
def outl(c, r1, r2):
    c = np.where(c>r1, random.randint(r1, r2), c)
    return c

#removing outliers for each column
data["BALANCE"] = outl(data["BALANCE"], int(data['BALANCE'].quantile(0.90)), int(data['BALANCE'].quantile(0.92)))
data["PURCHASES"] = outl(data["PURCHASES"], int(data['PURCHASES'].quantile(0.90)), 
                         int(data['PURCHASES'].quantile(0.91)))
data["ONEOFF_PURCHASES"] = outl(data["ONEOFF_PURCHASES"], int(data['ONEOFF_PURCHASES'].quantile(0.90)),
                                int(data['ONEOFF_PURCHASES'].quantile(0.91)))
data["INSTALLMENTS_PURCHASES"] = outl(data["INSTALLMENTS_PURCHASES"], int(data['INSTALLMENTS_PURCHASES'].quantile(0.90)),
                                      int(data['INSTALLMENTS_PURCHASES'].quantile(0.92)))
data["CASH_ADVANCE"] = outl(data["CASH_ADVANCE"], int(data['CASH_ADVANCE'].quantile(0.90)), 
                            int(data['CASH_ADVANCE'].quantile(0.91)))
data["PAYMENTS"] = outl(data["PAYMENTS"], int(data['PAYMENTS'].quantile(0.90)), int(data['PAYMENTS'].quantile(0.91)))
data["MINIMUM_PAYMENTS"] = outl(data["MINIMUM_PAYMENTS"], int(data['MINIMUM_PAYMENTS'].quantile(0.90)),
                               int(data['MINIMUM_PAYMENTS'].quantile(0.91)))

Elbow Plot and Clustering¶

In order to find the optimum value for k lets use elbow method which plots sum of squared error vs different values of k. Once the data has been clustered the labels are added to the original data frame in order to visualise the clusters.

kv = range(1,15)
for k in kv:
    kmc = KMeans(n_clusters=k)
    kmc.fit(dat2)
    sse.append(kmc.inertia_)

Observations¶

Cluster 0

  1. Has above average credit limit
  2. Uses the card for purchases quite frequently and is an above average spender.
  3. Uses it heavily on installment payments.
  4. Has paid heavily towards full amount.

Cluster 1

  1. Has below average credit limit.
  2. Uses the card less often and spending volume is below average.
  3. One among the least active users among the cluster.

Cluster 2

  1. Slightly more credit than cluster 2.
  2. Least frequent user with least amount of spending as well.

Cluster 3

  1. Has high credit limit.
  2. Normal purchases are quite infrequent and spending is low.
  3. Uses card primarily for cash advances.

Cluster 4

  1. Has the least credit.
  2. Uses card less often and spending volume is quite low.
  3. One among the least active users but slightly more active than cluster 1.
  4. Uses card primarily for cash advances but not as much as cluster 3.

Cluster 5

  1. Highest credit.
  2. Uses the card most often and spends the most among clusters. Power User.
